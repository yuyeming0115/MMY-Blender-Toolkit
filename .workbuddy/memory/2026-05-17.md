# 2026-05-17 工作日志

## Blender Toolkit - Catalog 分类乱码修复

**问题**：资产创建面板的 Catalog 分类下拉列表显示乱码（`*4`、`PE+`、`g-` 等），而 Blender 原生资产浏览器能正确显示中文分类名。

**根因分析（三层递进）**：
1. **第一层**：`split(":")` 无限制分割 → 改为 `split(":", 2)`，仍未解决
2. **第二层**：UUID 中含连字符 `-`，是 EnumProperty identifier 非法字符 → 添加 `_safe_enum_id`/`_unsafe_enum_id` hex 编码方案（参考 `mmy_3Dto2D_Sprites` 插件的 preset_service.py），仍未解决
3. **第三层（真正根因）**：手动读取 `blender_assets.cats.txt` 文件时编码解析错误，读出来的中文内容本身就是乱的。无论怎么处理 identifier 都无济于事。

**最终方案**：彻底放弃手动文件解析，改用 Blender 内置 **AssetCatalogTree API** 获取 catalogs（与原生资产浏览器同一数据源）。
- 优先从 FILE_BROWSER 空间的 `params.asset_library_catalogs` 获取
- 其次尝试 `bpy.asset.catalogs.catalog_tree` 等全局 API
- 回退：保留文件解析作为兜底

**修改文件**：
- `mmy_toolkit/asset_browser/properties.py`：完全重写 `get_catalog_enum_items()` 和新增 `_try_get_builtin_catalog_tree()`
- `mmy_toolkit/asset_browser/operators.py`：导入 `_unsafe_enum_id`，解码 catalog_enum
