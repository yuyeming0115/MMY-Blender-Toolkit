# 修改器图标获取问题分析

## 问题现象

自定义多列菜单无法显示修改器图标，而原生分类菜单（OBJECT_MT_modifier_add_generate 等）可以正确显示。

---

## 原因分析

### 原生菜单为什么能显示图标？

原生分类菜单（`OBJECT_MT_modifier_add_generate` 等）是在 **Blender C 层面**定义的。当它们调用 `operator("object.modifier_add")` 时：

1. Blender 内部通过 **RNA 系统**自动匹配修改器类型和图标
2. 这个匹配逻辑完全在 C 代码中实现
3. 不需要通过 Python API 传递图标参数

### 自定义菜单为什么无法获取图标？

尝试的方案及失败原因：

| 方案 | 失败原因 |
|------|----------|
| `bpy.types.ObjectModifierAdd.bl_rna` | Blender 5.1 中该 Python 类不存在 |
| `bpy.types.Operator.__subclasses__()` 遍历 | 只找到菜单类（OBJECT_MT_*），不是操作符类 |
| `Modifier.__subclasses__()` 遍历 | 无法获取操作符的 enum_items |
| `enum_items.icon` 返回值 | 可能不是有效的 icon_value |

### 根本原因

**Blender 没有把修改器图标的映射完全暴露给 Python API**。

- 修改器类型→图标的映射存在于 C 层面的 RNA 系统
- Python API 只能访问部分图标信息
- Blender 5.1 中部分图标名称字符串（如 MOD_ARRAY）也被移除或重命名

---

## 已尝试方案

1. **通过 Operator 类获取**：`ObjectModifierAdd` 不存在
2. **通过 Modifier 子类获取**：无 `icon` 属性或返回无效值
3. **通过 enum_items.icon 获取**：返回值不正确
4. **硬编码图标名称字符串**：`MOD_ARRAY` 等在 Blender 5.1 中不存在
5. **硬编码图标值（icon_value）**：无法获取正确值

---

## 可行方案

### 方案 A：使用原生分类菜单（已采用）

直接调用 Blender 原生分类菜单：
- `OBJECT_MT_modifier_add_generate`（生成）
- `OBJECT_MT_modifier_add_deform`（变形）
- `OBJECT_MT_modifier_add_edit`（修改）
- `OBJECT_MT_modifier_add_normals`（法向）
- `OBJECT_MT_modifier_add_physics`（物理）

**优点**：图标正确显示，无需额外处理
**缺点**：需要两级菜单（分类→修改器），不是单屏多列

### 方案 B：图标名称字符串映射（已验证失败）

尝试使用图标名称字符串（如 `MOD_ARRAY`）：
```python
op = col.operator("object.modifier_add", text="阵列", icon='MOD_ARRAY')
```

**失败原因**：Blender 5.1 已移除 `MOD_*` 系列图标名称。

**验证结果**：报错 `enum 'MOD_DECIMATE' not found`，确认所有 `MOD_*` 图标名称均不存在。

---

## 最终方案：使用原生分类菜单（已采用）

直接调用 Blender 原生分类菜单：
- `OBJECT_MT_modifier_add_generate`（生成）
- `OBJECT_MT_modifier_add_deform`（变形）
- `OBJECT_MT_modifier_add_edit`（修改）
- `OBJECT_MT_modifier_add_normals`（法向）
- `OBJECT_MT_modifier_add_physics`（物理）

**优点**：图标正确显示，无需额外处理
**缺点**：需要两级菜单（分类→修改器），不是单屏多列

---

## 技术参考

- Blender RNA 系统：在 C 层面维护类型和属性的映射
- `icon_value`：整数图标值，用于 `layout.operator(icon_value=N)`
- `icon`：字符串图标名，用于 `layout.operator(icon='MODIFIER_DATA')`
- 原生菜单定义：在 Blender 源码 `release/scripts/startup/bl_ui/properties_data_modifier.py`
