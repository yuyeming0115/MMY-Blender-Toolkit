# 智能命名模块

## 功能概述

解决 Blender 原生复制自动添加 `.001` 后缀的问题，提供智能复制和批量重命名功能。

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 智能复制集合 | 复制集合内容，名称数字递增（如 `集合1` → `集合2`） |
| 智能复制对象 | 复制对象，名称数字递增 |
| 批量重命名 | 前缀 + 后缀 + 序号递增 |
| 预设系统 | 常用前缀/后缀预设存储 |
| 烘焙分组 | 创建 `_low` 和 `_high` 子集合，用于烘焙工作流 |
| 快捷归组 | 将选中对象归组到新集合，支持命名对话框 |

## 预设系统

**存储文件**：`presets/naming_presets.json`

**默认预设**：
```json
{
  "separator": "_",
  "digits": 2,
  "prefix_presets": ["Model", "Char", "Prop", "Set", "Asset"],
  "suffix_presets": ["_01", "_LOD", "_A", "_B", "_High", "_Low"]
}
```

---

## 命名逻辑

```python
# 数字递增
smart_increment_name("Char_01", '_', 2) → "Char_02"
smart_increment_name("集合1", '', 1) → "集合2"
smart_increment_name("Model", '_', 2) → "Model_01"

# 批量重命名
apply_prefix_suffix("Body", "Char", "", 1, '_', 2) → "CharBody_01"
```

## 遇到的问题与解决方案

### 问题1：bpy_prop_collection.__contains__ 类型错误

**现象**：
- 复制对象时报错：`TypeError: bpy_prop_collection.__contains__: expected a string or a tuple of strings`
- 复制集合时报错：同上

**原因**：
```python
# 错误写法：直接使用对象判断是否在集合中
if obj in coll.objects:
    coll.objects.link(new_obj)

if source_coll in p.children:
    parent.children.link(new_coll)
```

Blender 的 `bpy_prop_collection`（如 `coll.objects`、`coll.children`）在使用 `in` 操作符时，**必须传入字符串（名称），而非对象本身**。

**解决方案**：
```python
# 正确写法：使用 .get() 方法或名称字符串
if coll.objects.get(obj.name):
    coll.objects.link(new_obj)

# 或者遍历查找
for child in p.children:
    if child.name == source_coll.name:
        parent.children.link(new_coll)
```

**关键点**：
- `bpy_prop_collection` 的 `in` 操作符只接受字符串参数
- 使用 `.get(name)` 方法检查对象是否存在
- 集合的 `children` 属性也是 `bpy_prop_collection`，同样适用此规则

### 问题2：N面板标签名不一致

**现象**：智能命名模块的 N面板显示为「MMY-Tools」，而其他模块统一显示为「MMY工具」

**原因**：
```python
# 错误写法
bl_category = "MMY-Tools"
```

**解决方案**：
```python
# 正确写法：统一使用中文标签
bl_category = "MMY工具"
```

### 问题3：偏好设置缺少预设编辑入口

**现象**：点击「配置预设」按钮打开偏好设置后，找不到前后缀预设的编辑区域

**原因**：偏好设置 draw() 函数中未添加预设编辑 UI

**解决方案**：在 `mmy_toolkit/__init__.py` 的 `MMY_Preferences.draw()` 中添加：

```python
# === 智能命名预设 ===
layout.label(text="智能命名预设:", icon='SORTBYEXT')
box = layout.box()

# 前缀预设编辑
from .smart_naming.presets import get_prefix_presets
prefix_presets = get_prefix_presets()
for prefix in prefix_presets:
    op = row.operator("mmy.remove_prefix_preset", text="", icon="X")
    op.prefix = prefix
    row.label(text=prefix)
row.operator("mmy.add_prefix_preset", text="添加前缀", icon="ADD")

# 后缀预设编辑（同理）
```

## 问题4：大纲右键菜单归组缺少命名弹窗

**现象**：从大纲右键菜单点击"归组到新集合"时，直接执行操作，未弹出命名对话框

**原因**：`layout.operator_context = 'INVOKE_DEFAULT'` 设置顺序错误，在添加 operator 之后才设置

**错误写法**：
```python
layout.operator("mmy.group_selected_objects")
layout.operator_context = 'INVOKE_DEFAULT'  # 设置在添加之后，无效
```

**正确写法**：
```python
layout.operator_context = 'INVOKE_DEFAULT'  # 先设置
layout.operator("mmy.group_selected_objects")  # 再添加
```

**关键点**：`operator_context` 必须在调用 `layout.operator()` **之前**设置，才能触发 operator 的 `invoke` 方法
