# 资产预览图批量刷新问题分析

## 问题现象
批量刷新资产预览图功能无效，无论是资产库缩略图还是文件管理器的 Blender 预览图都无法正确设置。

## 问题一：资产预览图 API 错误

### 错误的 API 使用
原代码 `_set_preview_direct` 方法使用了错误的 API：

```python
# ❌ 错误方式 - 这些 API 不适用于资产预览图
preview = bpy.data.previews.new()  # 创建的是 UI 自定义图标，不是资产预览图
preview.icon_pixels = byte_pixels
preview.image_pixels = byte_pixels
asset_id.asset_data.preview_image = preview  # preview_image 是只读属性，赋值无效
```

**问题分析：**
1. `bpy.data.previews.new()` 创建的是 **UI 自定义图标**（用于面板按钮等），不是资产预览图
2. `asset_data.preview_image` 属性是**只读的**，直接赋值不会生效
3. 即使代码执行不报错，预览图也不会真正保存到 .blend 文件中

### Blender 5.x API 变更
修正后的代码最初使用了旧版 override 方式：

```python
# ❌ Blender 4.x 方式 - 在 5.x 中不再支持
override = {'id': asset_id}
bpy.ops.ed.lib_id_load_custom_preview(override, filepath=preview_path)
```

**Blender 5.1 报错：**
```
_bpy.ops.call() argument 3 must be str, not dict
```

Blender 5.x 的 `bpy.ops` API 已改变，不再支持用 dict 作为 override 参数。

### 解决方案
```python
# ✅ Blender 5.x 正确方式
with bpy.context.temp_override(id=asset_id):
    bpy.ops.ed.lib_id_load_custom_preview(filepath=preview_path)
```

## 问题二：文件预览图（缩略图）不更新

### 问题现象
批量刷新后，Windows 文件管理器的 .blend 文件缩略图没有变化，显示为灰色模型或空白。

### 问题根因
1. **脚本打开文件时界面未渲染** - 直接保存时 Blender 没有捕获屏幕截图
2. **视窗模式不正确** - 默认是实体模式，不是材质预览模式
3. **模型没有最大化显示** - 缩略图显示不完整

### 解决方案：使用 Timer 延迟保存

**核心思路：** 打开文件 → 设置视窗 → 等待渲染 → 保存

```python
def _process_next_file(self):
    # 打开文件
    bpy.ops.wm.open_mainfile(filepath=blend_path)
    
    # 设置资产预览图
    self._set_preview_correct(asset_id, preview_path)
    
    # 切换视窗：材质预览 + 框显模型 + 最大化视窗
    self._setup_viewport_for_preview()
    
    # 注册 timer，延迟保存（等待界面渲染）
    bpy.app.timers.register(
        self._delayed_save,
        first_interval=0.5
    )

def _setup_viewport_for_preview(self):
    """切换 3D 视窗到材质预览模式，全选模型并最大化显示"""
    # 1. 切换到对象模式
    if bpy.context.mode != 'OBJECT':
        bpy.ops.object.mode_set(mode='OBJECT')
    
    # 2. 全选所有可见对象
    for obj in bpy.data.objects:
        if obj.visible_get():
            obj.select_set(True)
    visible_objs = [obj for obj in bpy.data.objects if obj.visible_get()]
    if visible_objs:
        bpy.context.view_layer.objects.active = visible_objs[0]
    
    # 3. 查找 3D 视窗并设置
    for window in bpy.context.window_manager.windows:
        for area in window.screen.areas:
            if area.type == 'VIEW_3D':
                space = area.spaces.active
                
                # 切换到材质预览模式
                space.shading.type = 'MATERIAL'
                
                # 框显所选 + 最大化视窗
                with bpy.context.temp_override(window=window, area=area, region=region, space_data=space):
                    bpy.ops.view3d.view_selected()
                    bpy.ops.screen.screen_full_area(use_hide_panels=True)
                return

def _delayed_save(self):
    """timer 回调：保存当前文件"""
    bpy.ops.wm.save_mainfile(filepath=blend_path)
    # 继续处理下一个文件
    self._process_next_file()
```

### 关键要点

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 切换对象模式 | 确保能正确全选 |
| 2 | 全选可见对象 | `obj.visible_get()` 过滤 |
| 3 | 设置活动对象 | `view_selected` 需要有目标 |
| 4 | 材质预览模式 | `space.shading.type = 'MATERIAL'` |
| 5 | 框显所选 | `bpy.ops.view3d.view_selected()` |
| 6 | 最大化视窗 | `bpy.ops.screen.screen_full_area(use_hide_panels=True)` |
| 7 | Timer 延迟 | 等待 0.5 秒让渲染完成 |
| 8 | 保存文件 | 此时截图已正确生成 |

## 完整处理流程

```
┌─────────────────────────────────────────────────────────────┐
│  批量刷新预览图                                              │
├─────────────────────────────────────────────────────────────┤
│  1. 收集所有 .blend 文件及预览图路径                          │
│  2. 打开第一个文件                                           │
│  3. 设置资产预览图（同名图片）                                 │
│  4. 切换视窗：材质预览 + 框显模型 + 最大化                     │
│  5. Timer 0.5秒 → 保存文件                                   │
│  6. Timer 0.2秒 → 打开下一个文件                              │
│  7. 重复步骤 3-6                                             │
│  8. 返回原文件，输出报告                                      │
└─────────────────────────────────────────────────────────────┘
```
