# 资产预览图批量刷新问题分析

## 问题现象
批量刷新资产预览图功能无效，无论是资产库缩略图还是文件管理器的 Blender 预览图都无法正确设置。

## 问题根因

### 错误的 API 使用
原代码 `_set_preview_direct` 方法使用了错误的 API：

```python
# ❌ 错误方式 - 这些 API 不适用于资产预览图
preview = bpy.data.previews.new()  # 创建的是 UI 自定义图标，不是资产预览图
preview.icon_pixels = byte_pixels
preview.image_pixels = byte_pixels
asset_id.asset_data.preview_image = preview  # preview_image 是只读属性，赋值无效
```

**问题分析：**
1. `bpy.data.previews.new()` 创建的是 **UI 自定义图标**（用于面板按钮等），不是资产预览图
2. `asset_data.preview_image` 属性是**只读的**，直接赋值不会生效
3. 即使代码执行不报错，预览图也不会真正保存到 .blend 文件中

### Blender 5.x API 变更
修正后的代码最初使用了旧版 override 方式：

```python
# ❌ Blender 4.x 方式 - 在 5.x 中不再支持
override = {'id': asset_id}
bpy.ops.ed.lib_id_load_custom_preview(override, filepath=preview_path)
```

**Blender 5.1 报错：**
```
_bpy.ops.call() argument 3 must be str, not dict
```

Blender 5.x 的 `bpy.ops` API 已改变，不再支持用 dict 作为 override 参数。

## 正确解决方案

### 使用 `temp_override()` 并传入 `id` 参数
```python
# ✅ Blender 5.x 正确方式
with bpy.context.temp_override(id=asset_id):
    bpy.ops.ed.lib_id_load_custom_preview(filepath=preview_path)
```

### 完整代码示例
```python
def _set_preview_correct(self, asset_id, preview_path):
    """使用正确的 API 设置资产预览图"""
    try:
        if not os.path.exists(preview_path):
            print(f"[MMY] 预览图文件不存在: {preview_path}")
            return False

        # 使用 temp_override 并传入 id 参数
        with bpy.context.temp_override(id=asset_id):
            bpy.ops.ed.lib_id_load_custom_preview(filepath=preview_path)
        print(f"[MMY] 预览图设置成功: {preview_path}")
        return True

    except Exception as e:
        print(f"[MMY] 设置预览图失败: {e}")
        return False
```

## 关键要点

| 场景 | 正确 API | 说明 |
|------|----------|------|
| 设置资产预览图 | `bpy.ops.ed.lib_id_load_custom_preview(filepath=...)` | 官方推荐方式 |
| 指定目标数据块 | `temp_override(id=asset_id)` | Blender 5.x 新语法 |
| UI 自定义图标 | `bpy.data.previews.new()` | 仅用于面板按钮等 UI 元素 |
