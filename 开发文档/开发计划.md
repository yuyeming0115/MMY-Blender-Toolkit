# MMY Blender Toolkit — 开发计划

## 项目概述

MMY Blender Toolkit 是一个统一的 Blender 效率优化插件集，通过模块化架构支持未来持续扩展。首个功能为「快速后缀保存」，在 Blender 文件浏览器的另存为对话框顶部注入后缀快捷按钮。

## 技术架构

```
MMY Blender Toolkit/
├── mmy_toolkit/                    # 插件根包（直接放 Blender addons 目录）
│   ├── __init__.py                 # 插件入口，统一注册/注销所有子模块
│   ├── config.py                   # 全局配置（偏好设置、默认后缀列表）
│   ├── utils.py                    # 公共工具函数（路径处理、文件名处理等）
│   │
│   ├── save_helper/                # 功能模块：快速后缀保存
│   │   ├── __init__.py             # 模块入口，注册当前模块的 operator/panel
│   │   └── operators.py            # SaveWithSuffix operator + 文件浏览器 header UI
│   │
│   ├── mesh_tools/                 # 功能模块：网格编辑工具
│   │   ├── __init__.py             # 模块入口
│   │   └── operators.py            # UV孤岛边界缝合边操作符
│   │
│   ├── asset_browser/              # 功能模块：资产创建工具
│   │   ├── __init__.py             # 模块入口
│   │   ├── operators.py            # 资产创建操作符
│   │   └── properties.py           # 属性组定义（路径、名称、Catalog等）
│   │
│   ├── rig_tools/                  # 功能模块：骨骼工具（预留）
│   │   ├── __init__.py
│   │   └── operators.py
│   │
│   └── ui/                         # UI 层：3D 视图侧边栏面板管理
│       ├── __init__.py             # 注册面板类
│       └── panels.py               # N面板 + 右键菜单挂载
```

## 关键设计决策

| 决策 | 理由 |
|------|------|
| **子包即模块** | 每个功能在独立子包下，`__init__.py` 只暴露 register/unregister，与主入口模式一致 |
| **`_modules` 列表控制开关** | 新增功能只需在列表中加一行，不需要改注册逻辑 |
| **`config.py` 统一管理偏好** | 所有模块共享的偏好设置集中一处，用 `bpy.props` 定义 |
| **`utils.py` 放公共函数** | 如路径清理、文件名处理等，避免各模块重复造轮子 |
| **`ui/` 仅管理 3D 侧边栏** | 文件浏览器 header 类 UI 放在各自功能模块中，不混在一起 |
| **bl_idname 统一前缀 `mmy`** | 如 `mmy.save_with_suffix`，保持命名空间一致 |
| **UI 融入官方界面** | 文件浏览器 header 注入（`FILEBROWSER_MT_header_append`），不创建额外侧边栏面板 |

## 模块划分

### 核心模块

| 模块 | 文件 | 职责 |
|------|------|------|
| `__init__.py` | 插件入口 | `bl_info` 声明，遍历 `_modules` 注册/注销 |
| `config.py` | 全局配置 | `bpy.props` 定义偏好设置（默认后缀列表等） |
| `utils.py` | 公共工具 | 文件名后缀智能替换、路径处理等 |

### 功能模块：save_helper

| 文件 | 职责 |
|------|------|
| `__init__.py` | 注册 `MMY_OT_SaveWithSuffix` operator，挂载 `draw_suffix_menu` 到文件浏览器 header |
| `operators.py` | 定义 `SaveWithSuffix` operator 和 `draw_suffix_menu` UI 绘制函数 |

**功能行为：**
- 后缀列表：`_Mesh`、`_Mat`、`_Rig`、`_Ani`、`_Render`
- 点击按钮 → 修改 `space_data.params.filename` → 对话框保持打开
- 智能去重：`scene_Mesh.blend` 再点击 `_Mesh` → 保持 `scene_Mesh.blend`

### Milestone 3：资产创建模块（已完成代码）

**功能概述：自动建立资产**
一键将选中的模型对象保存为Blender资产浏览器可识别的资产文件。

**功能需求：**
| 功能项 | 详细说明 |
|--------|----------|
| 资产路径 | 手动输入 + 自动记忆最近路径 + 收藏夹功能 |
| 资产名字 | 字符串输入，作为文件名和集合名 |
| 资产所属 | 使用现有资产库的Catalog分类系统 |
| 选中对象 | 支持多选，显示选中数量 |
| 建立资产按钮 | 执行核心操作 |

**核心操作流程：**
1. 在指定路径创建新.blend文件
2. 导入选中的模型对象
3. 创建集合（集合名=资产名）
4. 标记集合为资产并分配Catalog
5. 自动匹配同名png/jpg作为预览图
6. 记录使用路径到历史/收藏

**模块文件结构：**
```
mmy_toolkit/asset_browser/
├── __init__.py          # 模块入口（重写）
├── operators.py          # 操作符实现（新建）
└── properties.py         # 属性组定义（新建）
```

**功能概述：3D视图Header区域快速设置视图焦距**
