# 侧边栏管理器插件开发计划

## 项目概述

开发一个独立的 Blender 插件，用于管理 3D 视窗侧边栏（N 面板）中所有插件标签的排序顺序，增强 Blender 原生功能的不足。

## 问题分析

### Blender 原生排序机制

| 机制 | 说明 |
|------|------|
| 注册顺序 | 面板按 `bpy.utils.register_class()` 顺序注册 |
| 插件加载顺序 | 不同插件按偏好设置中的启用顺序加载 |
| bl_category | 同一 category 内的面板分组显示 |
| bl_order | 同一 category 内的面板排序（数值越小越靠前） |

**痛点**：用户无法自由调整侧边栏标签的顺序，必须通过禁用/重新启用插件来间接控制。

---

## 技术方案：启动时扫描 + 重排面板

### 核心思路

1. 在 `load_post` handler 中扫描所有已注册的 `Panel` 类
2. 提取每个面板的 `bl_category`、`bl_space_type`、`bl_region_type`
3. 根据用户配置的优先级，尝试重新触发面板显示

### 关键 API

```python
# 获取所有已注册的 Panel 类
for panel in bpy.types.Panel.__subclasses__():
    print(panel.bl_category, panel.bl_space_type, panel.bl_region_type)
```

### 实现流程

```
Blender 启动
    ↓
load_post handler 触发
    ↓
扫描所有已注册的 Panel 类
    ↓
提取 3D 视窗侧边栏面板（space_type='VIEW_3D', region_type='UI'）
    ↓
读取用户配置的标签优先级
    ↓
尝试重新触发面板显示顺序
    ↓
（如果无法直接控制，则记录并提示用户）
```

## 技术挑战与解决方案

### 挑战 1：无法直接修改其他插件的 bl_category

**问题**：`bl_category` 是类的静态属性，注册后无法动态修改。

**尝试方案**：
```python
# 尝试动态修改（可能失败）
panel.bl_category = "新名称"
```

**替代方案**：
- 通过 `bpy.types.Panel` 的 subclass 机制创建代理面板
- 或提示用户需要重启 Blender 并按新顺序启用插件

---

### 挑战 2：面板注册顺序由 C 端控制

**问题**：Python 无法干预 Blender C 端的注册顺序。

**替代方案**：
- 记录用户期望的顺序
- 在下次启动时，通过修改偏好设置中的插件启用顺序间接控制
- 或使用 `bpy.app.handlers.load_post` 在加载后重新"刷新"面板显示

---

### 挑战 3：确保稳定性

**解决方案**：
- 使用 `try/except` 包裹所有操作
- 不修改其他插件的代码，只读取信息
- 用户配置存储在独立文件中，不影响 Blender 原生偏好设置

## 核心代码实现

### 1. 扫描面板信息

```python
# panel_scanner.py

import bpy

def get_sidebar_panels():
    """获取所有 3D 视窗侧边栏面板"""
    panels = []
    for panel in bpy.types.Panel.__subclasses__():
        if hasattr(panel, 'bl_space_type') and hasattr(panel, 'bl_region_type'):
            if panel.bl_space_type == 'VIEW_3D' and panel.bl_region_type == 'UI':
                category = getattr(panel, 'bl_category', "未分类")
                if category:
                    panels.append({
                        'class': panel,
                        'category': category,
                        'bl_idname': panel.bl_idname,
                    })
    return panels

def get_unique_categories():
    """获取所有唯一的侧边栏标签"""
    panels = get_sidebar_panels()
    categories = set()
    for p in panels:
        categories.add(p['category'])
    return sorted(list(categories))
```

    """加载用户排序配置"""
    path = get_config_path()

### 3. 偏好设置面板

```python
# ui/preferences_panel.py

import bpy
from ..panel_scanner import get_unique_categories
from ..order_manager import load_order_config, save_order_config

class MMY_SIDEBAR_Prefs(bpy.types.AddonPreferences):
    bl_idname = "mmy_sidebar_manager"

    selected_category: bpy.props.StringProperty(default="")

    def draw(self, context):
        layout = self.layout

        # 获取所有标签
        categories = get_unique_categories()
        config = load_order_config()
        saved_order = config.get('order', [])

        # 按保存的顺序排列（未保存的按字母顺序）
        display_order = []
        for cat in saved_order:
            if cat in categories:
                display_order.append(cat)
        for cat in categories:
            if cat not in display_order:
                display_order.append(cat)

        # 绘制列表
        box = layout.box()
        box.label(text="侧边栏标签顺序：", icon='VIEW_3D')

        col = box.column(align=True)
        for i, cat in enumerate(display_order):
            row = col.row(align=True)
            row.label(text=f"  {i+1}. {cat}")

            # 上下移动按钮
            if i > 0:
                op_up = row.operator("mmy.sidebar_move_up", text="", icon='TRIA_UP')
                op_up.category = cat
            if i < len(display_order) - 1:
                op_down = row.operator("mmy.sidebar_move_down", text="", icon='TRIA_DOWN')
                op_down.category = cat

        # 提示
        layout.separator()
        layout.label(text="提示：重启 Blender 后生效", icon='INFO')
```
