# 雕刻工具模块开发记录

## 功能概述

为雕刻模式提供：
1. 右键菜单面组快捷操作
2. 悬浮 HUD 按钮系统（类似 MMY-FocusMode）

---

## 核心功能

### 1. 右键菜单面组操作

| 功能 | 操作符 | 说明 |
|------|--------|------|
| 从遮罩创建 | `sculpt.face_sets_create` (mode='MASKED') | 基于遮罩创建新面组 |
| 从可见面创建 | `sculpt.face_sets_create` (mode='VISIBLE') | 基于可见面创建 |
| 全部 | `sculpt.face_sets_create` (mode='ALL') | 为所有面创建面组 |
| 切换显示 | `sculpt.face_set_change_visibility` | 切换当前面组可见性 |
| 反转可见性 | `sculpt.face_set_invert_visibility` | 反转所有面组可见性 |
| 隐藏全部 | `sculpt.face_set_hide_all` | 隐藏所有面组 |
| 显示全部 | `sculpt.face_set_show_all` | 显示所有面组 |
| 填充 | `sculpt.face_set_edit` (mode='GROW') | 扩展面组范围 |
| 收缩 | `sculpt.face_set_edit` (mode='SHRINK') | 收缩面组范围 |
| 初始化面组 | `sculpt.face_set_initialize` (mode=...) | 根据不同条件初始化面组 |

**初始化面组 mode 参数**：

| mode 值 | 中文显示 |
|---------|----------|
| 'ALL' | 通用 |
| 'LOOSE_PARTS' | 按松散块 |
| 'BOUNDARY_EDGES' | 按边界边 |
| 'MATERIAL' | 按材质 |
| 'CREASE' | 按折痕 |
| 'SHARP_EDGES' | 按尖锐边 |
| 'UV_ISLANDS' | 按UV边界 |
| 'VISIBLE' | 按可见度 |

### 2. 悬浮 HUD 按钮

| 按钮 | 功能 | Blender 5.1 API |
|------|------|-----------------|
| 面组 | 切换面组显示 | `space.overlay.show_sculpt_face_sets` |
| 遮罩 | 切换遮罩显示 | `space.overlay.show_sculpt_mask` |
| 线框 | 切换线框显示 | `space.overlay.show_wireframes` |
| 把手 | 拖拽移动 HUD 位置 | 偏移值存于偏好设置 |
| + | 添加自定义按钮 | 待实现 |

### 问题 1：原生菜单类名错误

| 错误 | 正确 |
|------|------|
| `VIEW3D_MT_sculpt_context_menu` | `VIEW3D_PT_sculpt_context_menu` |

**原因**：Blender 5.1 中雕刻右键菜单是 **Panel** 类型（PT），不是 **Menu** 类型（MT）。

**验证方法**：`--debug-all` 启动 Blender，观察日志：
```
call wm.call_panel {'name': 'VIEW3D_PT_sculpt_context_menu'}
```

### 问题 4：Blender 5.1 GPU API 全面重构

这是最大的问题，Blender 5.1 的 GPU API 与 4.x 完全不同：

| API | Blender 4.x | Blender 5.1 |
|-----|-------------|--------------|
| Shader 名称 | `UNIFORM_COLOR_2D` | `UNIFORM_COLOR` |
| 设置 uniform | `shader.uniform_set("color", value)` | `shader.uniform_float("color", value)` |
| GPUVertBuf 参数 | `(属性数, 顶点数)` | `(GPUVertFormat, 顶点数)` |
| 创建顶点格式 | 不需要 | `GPUVertFormat()` + `attr_add(id, comp_type, len, fetch_mode)` |
| 索引缓冲 | `GPUVertBuf` | `GPUIndexBuf(type='TRIS', seq=indices)` |

**正确的 GPU 绘制代码（Blender 5.1）**：

```python
from gpu.types import GPUBatch, GPUVertBuf, GPUVertFormat, GPUIndexBuf
from gpu.state import blend_set

# 启用透明度混合
blend_set('ALPHA')

# 获取 shader
shader = gpu.shader.from_builtin('UNIFORM_COLOR')
shader.bind()
shader.uniform_float("color", (0.15, 0.15, 0.15, 0.85))

# 创建顶点格式
format = GPUVertFormat()
format.attr_add(id="pos", comp_type='F32', len=2, fetch_mode='FLOAT')

# 创建顶点缓冲
vbo = GPUVertBuf(format, len(vertices))
vbo.attr_fill("pos", vertices)

# 创建索引缓冲
ibo = GPUIndexBuf(type='TRIS', seq=indices)

# 创建批次并绘制
batch = GPUBatch(type='TRIS', buf=vbo, elem=ibo)
batch.draw(shader)
```

### 问题 6：雕刻模式下 context.area/context.region 为 None

**现象**：
- Object 模式：`context.area` 正常
- 雕刻模式：`context.area=None`, `context.region=None`

**原因**：雕刻模式改变了 Blender 的 context，modal operator 无法通过 `bpy.context` 获取正确的区域信息。

**解决方案**：使用鼠标坐标直接查找 area 和 region：

```python
def find_area_region_at_point(window, mouse_x, mouse_y):
    screen = getattr(window, "screen", None)
    if screen:
        for a in screen.areas:
            if (a.x <= mouse_x <= a.x + a.width and
                a.y <= mouse_y <= a.y + a.height):
                area = a
                for r in a.regions:
                    if (r.x <= mouse_x <= r.x + r.width and
                        r.y <= mouse_y <= r.y + r.height):
                        region = r
                        break
                return area, region
    return None, None
```

### 问题 7：指针比较失败（as_pointer）

**现象**：
- invoke 时存储：`area_id=2202576185696`
- modal 时查找：`a.as_pointer()=2203017600280`
- 比较失败：`2202576185696 != 2203017600280`

**原因**：Blender 内部会在某些操作后重新创建对象（虽然逻辑上是同一个 area），导致指针值变化。

**解决方案**：不能依赖 `as_pointer()` 比较，改用鼠标坐标范围查找（见问题 6）。

### 问题 10：多窗口拖拽不稳定

**现象**：
- 拖拽窗口 A 的 HUD，窗口 B 的 HUD 却移动了
- 两个窗口同时拖拽时，只有一个窗口有反应

**原因**：`_update_drag_position` 中使用 `context.window.as_pointer()` 获取窗口 ID，但雕刻模式下 context.window 可能变化，导致操作了错误的窗口偏移值。

**解决方案**：使用全局偏移值替代窗口独立存储，所有窗口同步（见问题 13）。

---

### 问题 11：Timer 机制过于复杂导致不触发

**现象**：Timer 注册成功但 `_ensure_hud_modal_running` 没有被调用。

**原因**：多个 Timer 函数（`_ensure_hud_modal_running`、`_periodic_hud_check`、`_delayed_start_modal`）相互冲突，`timer_registered` 标志管理混乱。

**解决方案**：简化为单一持久 Timer `_hud_modal_manager`，使用 `persistent=True`：

```python
def _hud_modal_manager():
    """单一 Timer：管理 HUD Modal 生命周期"""
    if not _HUD_STATE["enabled"]:
        return None  # 停止 Timer

    # 检查每个窗口，启动 Modal...
    return 2.0  # 每 2 秒检查一次

bpy.app.timers.register(_hud_modal_manager, first_interval=0.5, persistent=True)
```

---

### 问题 12：Modal 因窗口指针变化被终止

**现象**：Modal 启动成功但立即停止，按钮点击返回 `button_id=None`。

**原因**：modal() 方法中使用 `window.as_pointer() != self._window_id` 比较，Blender 会重新创建对象导致指针变化。

**解决方案**：改用 `modal_windows` 集合检查：

```python
def modal(self, context, event):
    # 不使用指针比较
    if self._window_id not in _HUD_STATE["modal_windows"]:
        return self._finish()
```

---

### 问题 13：按钮检测与绘制偏移值不一致

**现象**：点击 HUD 返回 `button_id=None`，但 HUD 正常显示。

**原因**：`find_button_at_point` 使用偏好设置的 `sculpt_hud_offset_x/y`，但 HUD 绘制使用 `window_offsets` 存储，两边不一致。

**解决方案**：统一使用全局偏移值 `get_global_offset()`。

---

### 问题 14：多窗口 HUD 位置不同步

**现象**：拖拽左侧窗口的 HUD，右侧窗口需要鼠标移过去才刷新。

**原因**：使用 `window_offsets` 存储每个窗口独立的偏移值。

**解决方案**：改为全局偏移值 `global_offset_x/y`，所有窗口同步：

```python
# hud_state.py
_HUD_STATE = {
    "global_offset_x": 0.0,
    "global_offset_y": 0.0,
}

def get_global_offset():
    return _HUD_STATE["global_offset_x"], _HUD_STATE["global_offset_y"]

def set_global_offset(offset_x, offset_y):
    _HUD_STATE["global_offset_x"] = offset_x
    _HUD_STATE["global_offset_y"] = offset_y
```

### 问题 15：对称功能不生效

**现象**：HUD 对称按钮高亮正常，但实际雕刻没有对称效果。

**原因**：Blender 有两种不同的对称系统：
- `sculpt.use_symmetry_x` - 镜像笔刷笔触（仅复制笔刷动作）
- `mesh.use_mirror_x` - **网格镜像**（真正的对称雕刻，Header 中激活的）

**解决方案**：使用 `mesh.use_mirror_x/y/z` 替代 `sculpt.use_symmetry_x/y/z`：

```python
# hud_modal.py
mesh = obj.data if obj and obj.type == 'MESH' else None
if mesh:
    mesh.use_mirror_x = not mesh.use_mirror_x

# hud_draw.py - 状态检查
mesh = obj.data if obj and obj.type == 'MESH' else None
if mesh:
    return bool(getattr(mesh, 'use_mirror_x', False))
```

### 问题 16：动态拓扑状态无法检测

**现象**：动态拓扑按钮功能正常（点击可开启/关闭），但高亮状态始终不变。

**原因**：Blender 5.1 API 不直接暴露动态拓扑激活状态：
- `mesh.use_dynamic_topology_sculpting` = None
- `sculpt.use_dyntopo` = None
- `sculpt.detail_type_method` = 'RELATIVE'（开/关都一样）

**解决方案**：使用手动状态跟踪，在 `_HUD_STATE` 中维护：

```python
# hud_state.py
_HUD_STATE = {
    "dyntopo_active": False,  # 手动跟踪
}

# hud_modal.py - 切换时更新
bpy.ops.sculpt.dynamic_topology_toggle()
_HUD_STATE["dyntopo_active"] = not _HUD_STATE.get("dyntopo_active", False)

# hud_draw.py - 状态检查
return _HUD_STATE.get("dyntopo_active", False)
```

### 问题 17：自动遮罩右键菜单不弹出

**现象**：右键点击自动遮罩按钮，菜单没有弹出。

**原因**：`MMY_OT_AutoMaskMenuModal.invoke` 重新调用 `find_button_at_point` 验证按钮位置，但由于事件处理时机问题，可能检测不到按钮，直接返回 `CANCELLED`。

**解决方案**：不再强制要求 `button_id == "auto_mask"`，改为使用鼠标位置作为 fallback：

```python
# hud_modal.py - MMY_OT_AutoMaskMenuModal.invoke
if layout_mode == "horizontal" and bounds and button_id == "auto_mask":
    # 精确计算按钮位置
    menu_x = btn_x
    menu_y = btn_y - menu_height
else:
    # fallback：鼠标位置附近弹出
    menu_x = region_mouse_x
    menu_y = region_mouse_y - menu_height
```
