# 资产预览图选中刷新功能开发计划

## 项目概述

在 MMY Blender Toolkit 插件的资产创建侧边栏中，新增「刷新预览图」区域，允许用户选择性地刷新指定资产的预览图，而非全量刷新整个目录。

## 功能需求

### 核心功能
1. 扫描指定目录下所有 .blend 文件
2. 显示文件列表，支持多选勾选
3. 显示每个文件的预览图状态（有/无对应预览图）
4. 批量刷新选中项的预览图

### 用户操作流程
```
┌─────────────────────────────┐
│  刷新预览图                  │
├─────────────────────────────┤
│  [扫描目录] [清空]           │
│                             │
│  [全选] [有预览] [无预览] [排除]│
│                             │
│  ☑ 模型A.blend  ✓ 有预览    │
│  ☐ 模型B.blend  🔴无预览    │
│  ☐ 模型C.blend  ✗ 已排除    │
│                             │
│  [刷新选中 (1)]             │
└─────────────────────────────┘
```

## 技术架构

### 新增文件

| `__init__.py` | 注册新增的 PropertyGroup 和 Operators |

### 数据结构

```python
# properties.py 新增属性
class MMY_RefreshPreviewFileItem(bpy.types.PropertyGroup):
    filepath: StringProperty()      # .blend 文件完整路径
    filename: StringProperty()      # 文件名（显示用）
    has_preview: BoolProperty()     # 是否有对应预览图
    preview_path: StringProperty()  # 预览图路径（如有）
    is_selected: BoolProperty()     # 是否选中

class MMY_AssetCreatorProps(bpy.types.PropertyGroup):
    # 现有属性...
    refresh_preview_files: CollectionProperty(type=MMY_RefreshPreviewFileItem)
    refresh_preview_index: IntProperty()
    excluded_files: CollectionProperty(type=MMY_RecentPath)  # 排除列表
```

### 新增 Operator

| Operator | 功能 |
|----------|------|
| `MMY_OT_ScanPreviewFiles` | 扫描目录，填充文件列表 |
| `MMY_OT_RefreshSelectedPreviews` | 刷新勾选的资产预览图 |
| `MMY_OT_ClearPreviewList` | 清空文件列表 |
| `MMY_OT_SelectAllPreviewFiles` | 全选/取消全选（切换逻辑） |
| `MMY_OT_SelectWithPreview` | 只选中有预览图的文件 |
| `MMY_OT_SelectNonePreview` | 只选中无预览图的文件 |
| `MMY_OT_ManageExcludedFiles` | 管理排除列表（添加、移除、清空） |

## UI 设计要点

### 视觉区分
- **有预览图**：正常显示，绿色勾选图标
- **无预览图**：红色高亮（`row.alert = True`），便于识别
- **已排除**：显示"已排除"标签，不参与刷新

### 快捷选择按钮
- **全选**：切换逻辑，点击时若已全选则取消全选，否则全选
- **有预览**：只选中有预览图的文件（排除无预览的）
- **无预览**：只选中无预览图的文件（用于识别缺失预览的资产）
- **排除**：弹出对话框管理排除列表，可添加、移除、清空

### 排除功能
- 排除列表存储在 `excluded_files` CollectionProperty 中
- 已排除的文件自动取消选中状态
- 排除列表在当前会话中保持（暂未持久化到配置文件）

## 技术问题记录

### 问题一：PropertyGroup 注册顺序
**现象**：`MMY_RefreshPreviewFileItem` 未注册导致 CollectionProperty 创建失败

**原因**：Blender 要求 CollectionProperty 引用的 PropertyGroup 必须先注册

**解决**：在 `asset_browser/__init__.py` 的 `_classes` 中，确保 `MMY_RefreshPreviewFileItem` 在 `MMY_AssetCreatorProps` 之前注册

```python
_classes = (
    properties.MMY_RecentPath,
    properties.MMY_FavoritePath,
    properties.MMY_RefreshPreviewFileItem,  # 必须在 MMY_AssetCreatorProps 之前
    properties.MMY_AssetCreatorProps,
    ...
)
```

### 问题二：Operator 未注册
**现象**：`UILayout.operator(): unknown operator 'mmy.scan_preview_files'`

**原因**：新增的 Operators 未添加到 `__init__.py` 的注册列表

**解决**：将所有新增 Operators 添加到 `_classes` 列表中

### 问题三：缺少导入
**现象**：`NameError: name 'StringProperty' is not defined`

**原因**：operators.py 中使用了 `StringProperty` 和 `EnumProperty` 但未导入

**解决**：在 operators.py 开头添加导入：
```python
from bpy.props import StringProperty, EnumProperty
```
