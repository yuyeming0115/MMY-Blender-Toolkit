# 自动备份功能开发记录

## 功能概述

为解决 Blender 5.x 自动保存失效问题，插件提供独立的自动备份机制。

## 备份策略

| 类型 | 间隔 | 文件名格式 | 滚动数量 |
|------|------|------------|----------|
| 小版本 | 默认 2 分钟（可配置） | `文件名_1430.blend` | 单日 20 个 |
| 大版本 | 默认 30 分钟（可配置） | `文件名_1430_MAJOR.blend` | 单日 20 个 |

## 技术要点

### 1. 定时器机制

使用 `bpy.app.timers` 实现定时触发：

```python
bpy.app.timers.register(
    callback,
    first_interval=60.0,  # 首次触发时间
    persistent=True       # 跨场景持久
)
```

### 2. 偏好设置存储

使用 `AddonPreferences` 定义属性，Blender 自动持久化：

```python
class MMY_Preferences(bpy.types.AddonPreferences):
    enabled_backup: bpy.props.BoolProperty(default=True)
    minor_interval_backup: bpy.props.IntProperty(default=2, min=1, max=60)
    major_interval_backup: bpy.props.IntProperty(default=30, min=5, max=120)
    # ...
```

### 3. 关键问题解决

#### 问题：register 期间 bpy.data 处于 restricted 状态

**现象**：在 `register()` 函数中访问 `bpy.data.filepath` 会报错：
```
'_RestrictData' object has no attribute 'filepath'
```

**解决方案**：使用定时器延迟启动，不依赖 register 期间的文件状态检查。定时器无条件启动，在回调中检查 `filepath` 是否存在。

#### 问题：persistent 定时器与模块变量状态不一致

**现象**：插件刷新（F8）后，模块级 `_timer_running` 变量重置为 `False`，但 persistent 定时器仍在运行，导致回调提前停止。

**解决方案**：移除 `_timer_running` 检查变量，定时器回调通过检查偏好设置的 `enabled_backup` 来决定是否继续运行。

### 4. 状态栏显示

挂载到 `STATUSBAR_HT_header`：

```python
bpy.types.STATUSBAR_HT_header.append(draw_statusbar_backup)
```

显示内容：
- 正常：`[MMY备份] 120MB | 下次: 14:32`
- 警告（超过阈值）：`[MMY备份] ⚠️ 520MB | 请清理`（红色）

### 5. 文件加载/保存处理

使用 handlers 重置时间戳，确保新文件/保存后能正确触发备份：

```python
bpy.app.handlers.load_post.append(_on_file_load)
bpy.app.handlers.save_post.append(_on_file_save)
```
