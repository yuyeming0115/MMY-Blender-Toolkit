# 应用修改器(保留形态键)开发计划

## 功能概述

Blender 原生"应用修改器"操作会删除形态键（Shape Keys），因为修改器可能改变顶点数量或拓扑结构。本功能旨在**应用修改器的同时保留形态键数据**。

## 技术背景

### 为什么原生应用会删除形态键？

形态键存储的是顶点相对位移数据。当修改器改变：
- **顶点数量**（如 Subsurf 增加顶点）→ 形态键索引失效
- **顶点顺序**（如 Mirror 改变拓扑）→ 形态键数据错乱
- **顶点位置**（如 Bevel）→ 基础形态变化

Blender 安全策略：直接删除形态键避免数据损坏。

## 实现方案（基于 SKkeeper 算法）

### 核心原理

参考官方插件 `blender_org/shapekey_keeper` 的实现方法：

1. **接收器对象**：复制原对象 → 只保留 Basis → 应用修改器
2. **捐献者对象**：对每个形态键，复制原对象 → 只保留该形态键 → 应用修改器
3. **使用 `join_shapes()` 传递形态键**：Blender 原生操作符，自动处理顶点匹配
4. **清理原对象，重命名接收器**

### 关键技巧

| 技术 | 说明 |
|------|------|
| `object.convert(target='MESH')` | 应用所有修改器，绕过形态键限制 |
| `join_shapes()` | 将源对象作为形态键合并到目标对象 |
| `modifier_apply()` | 应用单个修改器 |

### 核心流程

```python
def keep_shapekeys_apply_all(obj):
    # 1. 创建接收器（保留 Basis，应用修改器）
    receiver = copy_object(obj)
    apply_single_shapekey(receiver, 0)  # 只保留 Basis
    apply_modifiers_convert(receiver)   # 应用所有修改器

    # 2. 为每个形态键创建捐献者
    for sk_index in range(1, num_shapekeys):
        sk_donor = copy_object(obj)
        apply_single_shapekey(sk_donor, sk_index)
        apply_modifiers_convert(sk_donor)
        join_shapes(receiver, [sk_donor])  # 合并到接收器
        cleanup(sk_donor)

    # 3. 转移动画驱动器（如果有）
    transfer_drivers(obj, receiver)

    # 4. 清理原对象，重命名接收器
    cleanup(obj)
    receiver.name = obj.name
```
