# 资产预览图刷新功能开发记录

## 一、技术背景与问题分析

### 问题现象

批量刷新资产预览图功能无效，无论是资产库缩略图还是文件管理器的 Blender 预览图都无法正确设置。

---

### 问题一：资产预览图 API 错误

#### 错误的 API 使用

原代码 `_set_preview_direct` 方法使用了错误的 API：

```python
# ❌ 错误方式 - 这些 API 不适用于资产预览图
preview = bpy.data.previews.new()  # 创建的是 UI 自定义图标，不是资产预览图
preview.icon_pixels = byte_pixels
preview.image_pixels = byte_pixels
asset_id.asset_data.preview_image = preview  # preview_image 是只读属性，赋值无效
```

**问题分析：**
1. `bpy.data.previews.new()` 创建的是 **UI 自定义图标**（用于面板按钮等），不是资产预览图
2. `asset_data.preview_image` 属性是**只读的**，直接赋值不会生效
3. 即使代码执行不报错，预览图也不会真正保存到 .blend 文件中

#### Blender 5.x API 变更

修正后的代码最初使用了旧版 override 方式：

```python
# ❌ Blender 4.x 方式 - 在 5.x 中不再支持
override = {'id': asset_id}
bpy.ops.ed.lib_id_load_custom_preview(override, filepath=preview_path)
```

**Blender 5.1 报错：**
```
_bpy.ops.call() argument 3 must be str, not dict
```

Blender 5.x 的 `bpy.ops` API 已改变，不再支持用 dict 作为 override 参数。

#### 解决方案

```python
# ✅ Blender 5.x 正确方式
with bpy.context.temp_override(id=asset_id):
    bpy.ops.ed.lib_id_load_custom_preview(filepath=preview_path)
```

### 问题二：文件预览图（缩略图）不更新

#### 问题现象

批量刷新后，Windows 文件管理器的 .blend 文件缩略图没有变化，显示为灰色模型或空白。

#### 问题根因

1. **脚本打开文件时界面未渲染** - 直接保存时 Blender 没有捕获屏幕截图
2. **视窗模式不正确** - 默认是实体模式，不是材质预览模式
3. **模型没有最大化显示** - 缩略图显示不完整

#### 解决方案：使用 Timer 延迟保存

**核心思路：** 打开文件 → 设置视窗 → 等待渲染 → 保存

```python
def _process_next_file(self):
    # 打开文件
    bpy.ops.wm.open_mainfile(filepath=blend_path)
    
    # 设置资产预览图
    self._set_preview_correct(asset_id, preview_path)
    
    # 切换视窗：材质预览 + 框显模型 + 最大化视窗
    self._setup_viewport_for_preview()
    
    # 注册 timer，延迟保存（等待界面渲染）
    bpy.app.timers.register(
        self._delayed_save,
        first_interval=0.5
    )

def _setup_viewport_for_preview(self):
    """切换 3D 视窗到材质预览模式，全选模型并最大化显示"""
    # 1. 切换到对象模式
    if bpy.context.mode != 'OBJECT':
        bpy.ops.object.mode_set(mode='OBJECT')
    
    # 2. 全选所有可见对象
    for obj in bpy.data.objects:
        if obj.visible_get():
            obj.select_set(True)
    visible_objs = [obj for obj in bpy.data.objects if obj.visible_get()]
    if visible_objs:
        bpy.context.view_layer.objects.active = visible_objs[0]
    
    # 3. 查找 3D 视窗并设置
    for window in bpy.context.window_manager.windows:
        for area in window.screen.areas:
            if area.type == 'VIEW_3D':
                space = area.spaces.active
                
                # 切换到材质预览模式
                space.shading.type = 'MATERIAL'
                
                # 框显所选 + 最大化视窗
                with bpy.context.temp_override(window=window, area=area, region=region, space_data=space):
                    bpy.ops.view3d.view_selected()
                    bpy.ops.screen.screen_full_area(use_hide_panels=True)
                return

def _delayed_save(self):
    """timer 回调：保存当前文件"""
    bpy.ops.wm.save_mainfile(filepath=blend_path)
    # 继续处理下一个文件
    self._process_next_file()
```

#### 关键要点

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 切换对象模式 | 确保能正确全选 |
| 2 | 全选可见对象 | `obj.visible_get()` 过滤 |
| 3 | 设置活动对象 | `view_selected` 需要有目标 |
| 4 | 材质预览模式 | `space.shading.type = 'MATERIAL'` |
| 5 | 框显所选 | `bpy.ops.view3d.view_selected()` |
| 6 | 最大化视窗 | `bpy.ops.screen.screen_full_area(use_hide_panels=True)` |
| 7 | Timer 延迟 | 等待 0.5 秒让渲染完成 |
| 8 | 保存文件 | 此时截图已正确生成 |

### 完整处理流程

```
┌─────────────────────────────────────────────────────────────┐
│  批量刷新预览图                                              │
├─────────────────────────────────────────────────────────────┤
│  1. 收集所有 .blend 文件及预览图路径                          │
│  2. 打开第一个文件                                           │
│  3. 设置资产预览图（同名图片）                                 │
│  4. 切换视窗：材质预览 + 框显模型 + 最大化                     │
│  5. Timer 0.5秒 → 保存文件                                   │
│  6. Timer 0.2秒 → 打开下一个文件                              │
│  7. 重复步骤 3-6                                             │
│  8. 返回原文件，输出报告                                      │
└─────────────────────────────────────────────────────────────┘
```

## 二、选中刷新功能开发

### 功能需求

在资产创建侧边栏中，新增「刷新预览图」区域，允许用户选择性地刷新指定资产的预览图，而非全量刷新整个目录。

#### 核心功能

1. 扫描指定目录下所有 .blend 文件
2. 显示文件列表，支持多选勾选
3. 显示每个文件的预览图状态（有/无对应预览图）
4. 批量刷新选中项的预览图

#### 用户操作流程

```
┌─────────────────────────────┐
│  刷新预览图                  │
├─────────────────────────────┤
│  [扫描目录] [清空]           │
│                             │
│  [全选] [有预览] [无预览] [排除]│
│                             │
│  ☑ 模型A.blend  ✓ 有预览    │
│  ☐ 模型B.blend  🔴无预览    │
│  ☐ 模型C.blend  ✗ 已排除    │
│                             │
│  [刷新选中 (1)]             │
└─────────────────────────────┘
```

---

### 技术架构

#### 修改文件

| 文件 | 修改内容 |
|------|----------|
| `properties.py` | 新增 `refresh_preview_files` CollectionProperty、`excluded_files` 排除列表 |
| `operators.py` | 新增扫描目录、刷新选中资产、快捷选择、排除管理等 Operator |
| `panels.py` | 新增刷新预览图子面板 UI |
| `__init__.py` | 注册新增的 PropertyGroup 和 Operators |

#### 数据结构

```python
# properties.py 新增属性
class MMY_RefreshPreviewFileItem(bpy.types.PropertyGroup):
    filepath: StringProperty()      # .blend 文件完整路径
    filename: StringProperty()      # 文件名（显示用）
    has_preview: BoolProperty()     # 是否有对应预览图
    preview_path: StringProperty()  # 预览图路径（如有）
    is_selected: BoolProperty()     # 是否选中

class MMY_AssetCreatorProps(bpy.types.PropertyGroup):
    # 现有属性...
    refresh_preview_files: CollectionProperty(type=MMY_RefreshPreviewFileItem)
    refresh_preview_index: IntProperty()
    excluded_files: CollectionProperty(type=MMY_RecentPath)  # 排除列表
```

#### 新增 Operator

| Operator | 功能 |
|----------|------|
| `MMY_OT_ScanPreviewFiles` | 扫描目录，填充文件列表 |
| `MMY_OT_RefreshSelectedPreviews` | 刷新勾选的资产预览图 |
| `MMY_OT_ClearPreviewList` | 清空文件列表 |
| `MMY_OT_SelectAllPreviewFiles` | 全选/取消全选（切换逻辑） |
| `MMY_OT_SelectWithPreview` | 只选中有预览图的文件 |
| `MMY_OT_SelectNonePreview` | 只选中无预览图的文件 |
| `MMY_OT_ManageExcludedFiles` | 管理排除列表（添加、移除、清空） |

---

### UI 设计要点

#### 视觉区分

- **有预览图**：正常显示，绿色勾选图标
- **无预览图**：红色高亮（`row.alert = True`），便于识别
- **已排除**：显示"已排除"标签，不参与刷新

#### 快捷选择按钮

- **全选**：切换逻辑，点击时若已全选则取消全选，否则全选
- **有预览**：只选中有预览图的文件（排除无预览的）
- **无预览**：只选中无预览图的文件（用于识别缺失预览的资产）
- **排除**：弹出对话框管理排除列表，可添加、移除、清空

#### 排除功能

- 排除列表存储在 `excluded_files` CollectionProperty 中
- 已排除的文件自动取消选中状态
- 排除列表在当前会话中保持（暂未持久化到配置文件）

### 开发问题记录

#### 问题一：PropertyGroup 注册顺序

**现象**：`MMY_RefreshPreviewFileItem` 未注册导致 CollectionProperty 创建失败

**原因**：Blender 要求 CollectionProperty 引用的 PropertyGroup 必须先注册

**解决**：在 `asset_browser/__init__.py` 的 `_classes` 中，确保 `MMY_RefreshPreviewFileItem` 在 `MMY_AssetCreatorProps` 之前注册

```python
_classes = (
    properties.MMY_RecentPath,
    properties.MMY_FavoritePath,
    properties.MMY_RefreshPreviewFileItem,  # 必须在 MMY_AssetCreatorProps 之前
    properties.MMY_AssetCreatorProps,
    ...
)
```

#### 问题二：Operator 未注册

**现象**：`UILayout.operator(): unknown operator 'mmy.scan_preview_files'`

**原因**：新增的 Operators 未添加到 `__init__.py` 的注册列表

**解决**：将所有新增 Operators 添加到 `_classes` 列表中

#### 问题三：缺少导入

**现象**：`NameError: name 'StringProperty' is not defined`

**原因**：operators.py 中使用了 `StringProperty` 和 `EnumProperty` 但未导入

**解决**：在 operators.py 开头添加导入：
```python
from bpy.props import StringProperty, EnumProperty
```
