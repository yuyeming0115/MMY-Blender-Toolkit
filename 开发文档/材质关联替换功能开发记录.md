# 材质关联替换功能开发记录

## 功能概述

从外部 Blender 文件 **Link** 材质球到当前场景，建立映射关系后批量替换场景中所有对象的材质。

---

## 一、材质替换功能

### 交互流程

```
选择外部文件 → 勾选要Link的材质 → Link并生成映射 → 调整映射 → 执行替换
```

### 自动匹配规则

1. **完全同名匹配**：材质名（去掉数字后缀后）完全相同
2. **后缀匹配**：后缀相同（如 `Body`、`Weapon`、`Head`）但前缀不同
3. **未匹配**：显示"不替换"，用户可手动选择

---

## 二、关联动画功能

### 功能说明

从外部 Blender 文件 Link 名为 **Ani** 的集合到当前场景，并放置在世界中心。

### 交互流程

```
选择动画文件 → 检查Ani集合 → 关联到世界中心
```

### 关键 API

```python
# Link 集合
with bpy.data.libraries.load(filepath, link=True) as (data_from, data_to):
    data_to.collections = ["Ani"]

# 创建集合实例
ani_collection = bpy.data.collections.get("Ani")
instance = bpy.data.objects.new("Ani", None)
instance.empty_display_type = 'PLAIN_AXES'  # 空对象显示样式（坐标轴）
instance.instance_type = 'COLLECTION'       # 实例类型为集合
instance.instance_collection = ani_collection
instance.location = (0, 0, 0)  # 世界中心
context.scene.collection.objects.link(instance)
```

**注意**：`empty_display_type` 和 `instance_type` 是两个不同属性：
- `empty_display_type`：空对象的视觉显示样式（坐标轴、箭头、圆等）
- `instance_type`：实例的数据类型（集合、物体等）

## 三、骨骼缩放控制功能

### 功能说明

给场景中的 Armature 骨骼添加 Copy Scale 约束，通过 Scale 空物体控制骨骼整体缩放。

### 交互流程

```
选择目标骨骼 → 创建约束 → 调整缩放值 → 启用/禁用约束
```

### UI布局

```
┌─────────────────────────────────────┐
│ [骨骼缩放]                            │
├─────────────────────────────────────┤
│ 骨骼: [下拉选择] [创建约束] ✓          │
│                                      │
│ 缩放: [滑块] (1.00)                   │
│ ☑ 偏移  叠加原有缩放                   │
└─────────────────────────────────────┘
```

### 关键 API

```python
# 创建 Copy Scale 约束
constraint = armature.constraints.new('COPY_SCALE')
constraint.name = "MMY_Copy_Scale"
constraint.target = scale_obj  # Scale 空物体
constraint.use_x = True
constraint.use_y = True
constraint.use_z = True
constraint.use_offset = True  # 偏移模式（叠加原有缩放）
constraint.owner_space = 'LOCAL'
constraint.target_space = 'LOCAL'
constraint.mute = False  # 启用/禁用约束
```

**约束参数说明**：
- `use_offset`：叠加在原有缩放上，而不是直接覆盖
- `mute`：True = 禁用约束，False = 启用约束

## 四、遇到的问题与解决方案

### 问题1：Link材质和本地材质同名导致替换失败

**现象**：
- 映射关系正确建立
- 点击替换后显示"已是目标材质，跳过"
- 模型对象的材质球没有变化

**原因**：
```python
# 错误写法：同名时 get() 返回本地材质，不是 Link 材质
target_mat = bpy.data.materials.get(target_name)
```

当源材质名 `502016_Body` 和目标材质名 `502016_Body` 完全相同时，`bpy.data.materials.get()` 会返回本地材质对象，而不是 Link 进来的材质。

**解决方案**：
```python
# 正确写法：通过 library 属性区分 Link 材质
target_mat = None
for mat in bpy.data.materials:
    if mat.name == target_name and mat.library is not None:
        target_mat = mat
        break
```

**关键点**：
- Link 材质的 `mat.library` 属性不为 `None`
- 本地材质的 `mat.library` 属性为 `None`
- 替换时需要比较材质**对象引用**，而不是名字

### 问题2：属性回调函数定义顺序错误

**现象**：
- 插件加载时报错：`NameError: name '_update_constraint_enabled' is not defined`
- 类定义中的 `update=_update_constraint_enabled` 找不到函数

**原因**：
```python
# 错误写法：函数定义在类后面
class MMY_MatReplacerProps(bpy.types.PropertyGroup):
    constraint_enabled: BoolProperty(
        update=_update_constraint_enabled  # 此时函数还未定义！
    )

def _update_constraint_enabled(self, context):  # 定义在类后面
    ...
```

Python 在类定义时会立即解析所有属性，此时 `_update_constraint_enabled` 函数尚未定义，导致报错。

**解决方案**：
```python
# 正确写法：函数定义在类前面
def _update_constraint_enabled(self, context):
    ...

class MMY_MatReplacerProps(bpy.types.PropertyGroup):
    constraint_enabled: BoolProperty(
        update=_update_constraint_enabled  # 函数已定义，可正常引用
    )
```

**关键点**：
- 所有作为 `update` 回调的函数必须在类定义**之前**声明
- 这是 Blender PropertyGroup 的常见陷阱

### 问题3：Git仓库与Blender插件目录文件不同步

**现象**：
- Git仓库代码已修复，但Blender仍报相同错误
- `git push` 后插件仍无法加载

**原因**：
Blender 插件目录 (`C:\BlenderAPP\...\scripts\addons\`) 和 Git 仓库 (`D:\GitWork\...`) 是两个独立位置，修改Git仓库不会自动更新Blender目录。

**解决方案**：
- 手动同步：修改后需要同时更新两处文件
- 或使用脚本自动同步
- 推送前确认 Blender 插件目录的文件也是最新版本

## 六、FBX导入增强

### 新增功能

在FBX导入工具中新增"清零变换"选项，导入后自动清除模型的位移、缩放、旋转。

### UI布局

```
┌─────────────────────────────────────┐
│ [导入工具]                            │
├─────────────────────────────────────┤
│ [导入FBX] ☑动画                       │
│ ☑引用材质 ☑清零变换                   │
└─────────────────────────────────────┘
```

### 关键代码

```python
# 属性定义（mesh_tools/__init__.py）
bpy.types.Scene.mmy_clear_transforms = bpy.props.BoolProperty(
    name="清零变换",
    description="导入后清除模型的位移、缩放、旋转",
    default=True
)

# 清零方法（mesh_tools/operators.py）
def _clear_transforms(self, new_objects):
    for obj in new_objects:
        obj.location = (0, 0, 0)
        obj.rotation_euler = (0, 0, 0)
        obj.scale = (1, 1, 1)
```
