# 材质关联替换功能开发记录

## 功能概述

从外部 Blender 文件 **Link** 材质球到当前场景，建立映射关系后批量替换场景中所有对象的材质。

## 设计方案

### 交互流程

```
选择外部文件 → 勾选要Link的材质 → Link并生成映射 → 调整映射 → 执行替换
```

### UI布局（双列）

| 左列 | 右列 |
|------|------|
| 外部材质勾选列表 | 映射列表（场景材质名 + 下拉选择框） |

**映射列表设计**：
- 左侧显示场景材质名
- 右侧是下拉选择框，可选择 Link 进来的材质
- 自动匹配会填充下拉框，用户可手动修改

### 自动匹配规则

1. **完全同名匹配**：材质名（去掉数字后缀后）完全相同
2. **后缀匹配**：后缀相同（如 `Body`、`Weapon`、`Head`）但前缀不同
3. **未匹配**：显示"不替换"，用户可手动选择

## 遇到的问题与解决方案

### 问题1：Link材质和本地材质同名导致替换失败

**现象**：
- 映射关系正确建立
- 点击替换后显示"已是目标材质，跳过"
- 模型对象的材质球没有变化

**原因**：
```python
# 错误写法：同名时 get() 返回本地材质，不是 Link 材质
target_mat = bpy.data.materials.get(target_name)
```

当源材质名 `502016_Body` 和目标材质名 `502016_Body` 完全相同时，`bpy.data.materials.get()` 会返回本地材质对象，而不是 Link 进来的材质。两者的 `name` 属性相同，但对象引用不同。

**解决方案**：
```python
# 正确写法：通过 library 属性区分 Link 材质
target_mat = None
for mat in bpy.data.materials:
    if mat.name == target_name and mat.library is not None:
        target_mat = mat
        break
```

**关键点**：
- Link 材质的 `mat.library` 属性不为 `None`
- 本地材质的 `mat.library` 属性为 `None`
- 替换时需要比较材质**对象引用**，而不是名字

### 问题2：EnumProperty 动态列表中文显示乱码

**解决方案**：使用项目已有的 `_safe_enum_id()` 模式：

```python
_TARGET_MAT_CACHE = []

def get_target_mat_items(self, context):
    _TARGET_MAT_CACHE.clear()
    _TARGET_MAT_CACHE.append(("none", "不替换", ""))

    for mat in bpy.data.materials:
        if mat.library is not None:
            display = mat.name.split('@')[0]  # 清理库名标识
            safe_id = 'c' + mat.name.encode('utf-8').hex()  # UTF-8 hex编码
            _TARGET_MAT_CACHE.append((safe_id, display, ""))

    return _TARGET_MAT_CACHE
```
