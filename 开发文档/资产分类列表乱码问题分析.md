# 资产分类列表乱码问题分析与修复

**日期**：2026-05-17  
**状态**：已修复

## 问题描述

"分类"下拉列表显示乱码（如 `@اس`、`Pاس` 等阿拉伯字符），多次尝试修复无效。

## 根本原因

乱码不是文件读取编码问题，而是 **EnumProperty `items` 元组的 description 字段传入了含中文的路径字符串**。

Blender EnumProperty items 格式：`(identifier, name, description)`

原代码将 `catalog_path`（如 `Game/轿跑钥匙`）作为 description 传入：
```python
items.append((safe_id, catalog_simple, catalog_path))  # 错误
```

Blender 在 Windows 上以系统代码页（GBK/CP936）渲染 description 字段，而字符串是 UTF-8，导致中文字节被错误解释为阿拉伯字符（Windows-1256 特征）。

`name` 字段（显示名称）不受影响，因为 Blender 对其有正确的 Unicode 处理。

## 修复方案

将两处 `items.append` 的 description 改为空字符串：

**文件**：`mmy_toolkit/asset_browser/properties.py`

| 位置 | 修改前 | 修改后 |
|------|--------|--------|
| 第 63 行（内置API路径） | `(safe_id, node.display_name, node.path)` | `(safe_id, node.display_name, "")` |
| 第 113 行（文件解析路径） | `(safe_id, catalog_simple, catalog_path)` | `(safe_id, catalog_simple, "")` |
